# AI Coding Playbook

面向产品经理、UI/UX 设计师、运营、业务分析师等非开发角色的软件工程意识教程。

本项目不教你从零写代码，也不整理提示词技巧。它关注的是：非开发人员如何在 AI coding 中理解反馈、判断问题、验证结果，并把一次次经验沉淀下来。

基础的软件工程意识能把零散的报错、修改和反馈转化为可积累的经验；如果缺少这层意识，很多问题只会停留在一次性修复里，难以系统化沉淀。

## 默认前提

本教程默认你已经有 GitHub 账号，并且可以使用 Claude Code、Codex 或同类 AI coding 工具。本项目不讲这些工具的注册、安装和基础使用。

## 本项目聚焦什么

本项目帮助非开发角色建立一套最小的软件工程意识：

- 看懂 AI coding 中常见的反馈；
- 理解需求、任务、Issue、规格、测试、提交等基础概念；
- 知道如何让 AI 获得可验证的反馈；
- 看懂一次代码改动大致影响了什么；
- 把问题处理过程沉淀为检查清单、提示词和项目规则。

教程固定围绕一个极简的“个人库存订单管理工具”展开：创建商品和规格、手动入库、创建订单、提交后扣减库存、退单后恢复库存。这个案例只是教学载体，重点不是业务本身，而是 AI coding 中的工程判断。

## 本项目不覆盖什么

软件还包括用户体验、交互设计、视觉设计、信息架构、可访问性等重要内容。这些都很重要，但不是本项目重点。

本教程中的页面只要求能看见、能操作、能验证，不追求完整 UI/UX 质量；也不替代编程语言教程、框架教程、计算机科学课程或专业开发者工程手册。

## 学习方式

本项目采用场景串联和螺旋式学习。概念不会一次讲透，而是在任务中反复出现：先知道它存在，再通过 AI 对话、验证反馈和复盘逐步理解。

每章按这个顺序阅读：

1. `README.md`：看本章大纲；
2. `lesson.md`：完成学习内容、提示词对话和本章任务；
3. `self-check.md`：做自我验证和复盘。

课程大纲见：[docs/curriculum-outline.md](docs/curriculum-outline.md)。

## 适合谁

- 想用 AI coding 做原型、工具或内部系统的产品经理；
- 希望理解工程反馈和实现边界的 UI/UX 设计师；
- 需要与工程团队更高效沟通的软件公司非开发角色；
- 想把 AI coding 经验持续沉淀成个人或团队知识库的人。
