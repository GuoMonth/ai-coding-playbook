# 自我验证

## 完成清单

- [ ] 我创建或准备创建了最小测试与回归验证需求 Issue；
- [ ] 需求 Issue 写清楚了优先测试哪些关键规则；
- [ ] 需求 Issue 写清楚了测试如何运行；
- [ ] 我创建或准备创建了测试开发 Issue；
- [ ] 开发 Issue 引用了需求 Issue；
- [ ] 项目有少量可运行测试；
- [ ] 测试覆盖了库存、订单或表单中的关键规则；
- [ ] 我运行过测试命令；
- [ ] 我知道测试失败时应该把失败信息反馈给 AI；
- [ ] 本章没有追求全量覆盖、端到端测试体系或 CI/CD；
- [ ] 本章没有新增业务功能、大范围重构、后端服务、真实数据库、登录、权限或多租户；
- [ ] 我看过本章 diff，并大致理解哪些改动是测试相关。

## 自检问题

如果你能回答下面的问题，就可以进入下一章：

1. 测试为什么能帮助 AI coding？
2. 什么是回归？
3. 为什么本章只选少量高价值测试？
4. 测试失败时，应该把什么信息给 AI？
5. 哪些场景仍然适合人工看页面验证？
6. 如果 AI 为了写测试顺手改业务规则，应该怎么处理？
7. 这次 diff 中哪些文件是测试，不是业务功能？

```text
测试不是额外负担，而是让 AI 获得清晰反馈并持续修正的工具。
```
