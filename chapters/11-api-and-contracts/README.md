# 11. API And Contracts

本章目标是理解页面和数据服务之间的接口契约。

完成本章后，你应该知道：

- 页面不应该永远直接关心数据怎么保存；
- API 契约要说明请求、响应、字段和错误；
- mock 可以先模拟真实服务；
- 契约不清会让 AI coding 更容易跑偏；
- 前后端协作的核心不是“谁写代码”，而是“约定是否清楚”。

## 背景

前面章节可以直接在页面里处理数据。随着商品、库存和订单逻辑增多，页面和数据操作会越来越纠缠。

本章不要求搭建完整后端，只让学习者理解接口契约是什么，以及为什么它能帮助 AI coding。

## 实现边界

本章可以使用 mock 或最小数据服务示例。

本章明确不做：

- 完整后端服务；
- 真实数据库；
- 登录、权限、多租户；
- 微服务、网关、鉴权体系；
- 性能优化和缓存体系；
- 大规模重写已有功能。

## 本章最终结果

完成本章后，项目应该有一份最小接口契约，至少覆盖：

- 获取商品列表；
- 创建商品；
- 提交订单；
- 退回订单；
- 成功响应字段；
- 错误响应字段；
- mock 或最小服务如何按契约返回数据。

页面是否全部改造为调用服务，可以按最小范围推进，不要求一次重写。
