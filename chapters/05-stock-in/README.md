# 05. Stock In

本章目标是实现手动入库，让商品库存增加。

完成本章后，你应该知道：

- 一个操作会改变数据状态；
- 数量字段需要边界条件；
- 库存变化需要能被验证；
- 入库和订单扣减是两个不同流程；
- 简单库存流水可以帮助理解“发生过什么”。

## 背景

前面章节已经有商品和规格，但库存只是一个展示字段。本章开始让库存发生变化。

这里的入库是最简单的手动入库：选择商品，输入数量，确认后库存增加。

## 实现边界

本章只做库存增加。

本章明确不做：

- 订单提交和库存扣减；
- 退单和库存恢复；
- 库存预占、冻结、调拨；
- 完整库存流水系统；
- 数据持久化；
- API、数据库、登录、权限、多租户；
- 并发、幂等、一致性深入讨论。

## 本章最终结果

完成本章后，页面应该支持一次手动入库。最小流程包括：

- 选择一个商品；
- 输入入库数量；
- 数量必须大于 0；
- 提交后商品库存增加；
- 页面能看到变化前后的库存；
- 可以显示一条最近入库记录或操作提示。

数据可以继续保存在页面内临时状态中。
