# 01. First Visible Page

本章的目标是让项目先能跑起来，再做出第一个可见页面。

完成本章后，你应该知道：

- 初始化项目本身就是一个独立任务；
- 需求讨论和开发实现可以拆成不同 Issue；
- 开发 Issue 可以引用需求 Issue；
- AI coding 应该先按规格和检查点执行，再根据反馈修正；
- 页面能看到，不等于真实业务功能已经完成。

## 背景

第 0 章建立了最小 AI coding 工作方式。本章开始进入第一个实际任务：让“个人库存订单管理工具”从空仓库变成一个能运行、能看到首页的前端项目。

本章会拆成三个 Issue：

1. 初始化项目；
2. 讨论首页需求；
3. 实现首页。

这样做是为了让你先体会一个重要区别：有些 Issue 是搭环境，有些 Issue 是讨论需求，有些 Issue 才是真正开发实现。

## 实现边界

本章只允许做“项目能运行”和“首页能看见”。不要让 AI 顺手扩展功能。

本章明确不做：

- 登录、注册、身份验证；
- 用户、权限、多租户；
- 商品列表、商品表单；
- 库存入库、扣减、恢复；
- 订单创建、提交、退单；
- 数据库、本地存储、API；
- 路由、侧边栏、复杂导航；
- 自动化测试。

## 默认技术方案

本章直接使用：

- Vite；
- React；
- TypeScript。

你不需要自己选择框架。这个方案只是为了让项目快速启动、快速看到页面、快速获得 AI coding 反馈。

测试工具会在后续章节逐步引入。本章先以页面能启动、能看到指定内容为主要验证方式。

## 本章最终结果

完成本章后，项目应该可以在本地启动，并看到一个首页。首页至少包含：

- 标题：个人库存订单管理工具；
- 一句说明：用于记录商品、库存和订单；
- 三个入口占位：商品、库存、订单；
- 一个提示：当前还没有接入真实数据。
