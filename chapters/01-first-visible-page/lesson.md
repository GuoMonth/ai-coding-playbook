# 学习

本章用三个小 Issue 完成第一个可见页面：先初始化项目，再讨论首页需求，最后按规格实现首页。

## 1. Issue 1：初始化项目

第一个 Issue 只做项目初始化，不做业务页面。

复制下面的提示词给 AI：

```text
请通过 gh CLI 创建一个 GitHub Issue，用来跟踪“初始化个人库存订单管理工具前端项目”。

Issue 范围：

- 使用教程默认技术方案：Vite + React + TypeScript；
- 初始化一个能运行的前端项目；
- 能安装依赖；
- 能启动开发服务器；
- 能看到默认页面或极简占位页面；
- 不实现商品、库存、订单功能；
- 不设计首页业务内容；
- 不增加登录、权限、路由、API、数据库、本地存储或测试框架。

请在 Issue 中写出：

- 最小规格；
- 验证检查点；
- 明确的“本次不做”。

创建前请先给我看标题和正文，等我确认后再运行 gh 命令。
```

确认 Issue 后，让 AI 执行：

```text
请根据刚才的初始化项目 Issue 执行开发任务。

要求：

- 按 Issue 中的规格初始化 Vite + React + TypeScript 项目；
- 不实现业务首页；
- 不增加登录、权限、路由、API、数据库、本地存储或测试框架；
- 完成后运行必要检查，确认项目能启动；
- 如果失败，先解释错误属于哪类问题，再修复；
- 最后说明创建或修改了哪些文件。
```

这个 Issue 完成后，你应该至少知道：项目初始化和业务功能开发不是一回事。

## 2. Issue 2：讨论首页需求

第二个 Issue 只讨论首页需求，不直接开发。

复制下面的提示词给 AI：

```text
请通过 gh CLI 创建一个 GitHub Issue，用来讨论“个人库存订单管理工具首页需求”。

这个 Issue 只做需求讨论，不直接开发。

请在 Issue 中记录：

- 首页要表达什么；
- 页面上应该出现哪些信息；
- 商品、库存、订单三个入口只是占位，不做真实功能；
- 当前不接入真实数据；
- 不讨论登录、权限、多租户、数据库、API、订单规则或库存规则；
- 需要形成最终规格和验证检查点；
- 后续会创建单独的开发 Issue 来实现。

创建前请先给我看标题和正文，等我确认后再运行 gh 命令。
```

然后让 AI 帮你收敛需求：

```text
请基于这个首页需求 Issue，先不要写代码。

请和我讨论并收敛：

1. 首页最小应该展示哪些内容；
2. 哪些内容本章不做；
3. 最终规格；
4. 验证检查点；
5. 后续开发 Issue 应该如何引用这个需求 Issue。

边界要求：

- 不引入登录、注册、用户、权限、多租户；
- 不引入商品、库存、订单的真实功能；
- 不讨论数据库、API、本地存储；
- 不设计复杂导航和多页面结构。

请用非开发人员能理解的语言输出。
```

这个 Issue 完成后，你应该看到：需求讨论可以独立存在，它的产出是规格和检查点，而不一定马上改代码。

## 3. Issue 3：实现首页

第三个 Issue 才开始实现首页。它应该引用第二个需求 Issue。

复制下面的提示词给 AI：

```text
请通过 gh CLI 创建一个 GitHub Issue，用来实现“个人库存订单管理工具首页”。

要求：

- 引用前一个首页需求 Issue；
- 按需求 Issue 中确认的规格实现；
- 首页显示“个人库存订单管理工具”；
- 首页显示一句说明：用于记录商品、库存和订单；
- 首页显示三个入口占位：商品、库存、订单；
- 首页显示“当前还没有接入真实数据”的提示；
- 不实现商品、库存、订单真实功能；
- 不增加登录、权限、多租户、路由、API、数据库、本地存储；
- 完成后运行项目并按验证检查点检查。

创建前请先给我看标题和正文，等我确认后再运行 gh 命令。
```

确认 Issue 后，让 AI 实现：

```text
请根据首页开发 Issue 执行实现。

要求：

- 开发前先复述最终规格和验证检查点；
- 按规格修改文件；
- 只改实现首页所需的最小文件；
- 不增加登录、权限、路由、API、数据库、本地存储或自动化测试；
- 运行项目或必要检查；
- 如果页面打不开，把错误分类并修复；
- 完成后说明这次创建或修改了哪些文件。
```

## 4. 观察页面

本章最重要的不是页面好不好看，而是你第一次看到：

```text
初始化项目 -> 讨论需求 -> 实现页面 -> 运行项目 -> 浏览器看到页面 -> 按检查点验证
```

如果页面打不开，把错误信息交给 AI，让它先解释错误属于哪一类，再修复。

## 5. 看 diff

页面能打开后，让 AI 解释这次改动：

```text
请查看本章所有相关 diff，用非开发人员能理解的方式说明：

- 初始化项目改了哪些文件；
- 首页实现改了哪些文件；
- 哪些文件和页面显示有关；
- 哪些文件和项目运行有关；
- 我现在不需要理解哪些细节。
```

你不需要读懂所有代码，但要开始形成一个习惯：每次 AI coding 后都要看这次到底改了什么。
