# 自我验证

## 完成清单

- [ ] 我创建或准备创建了初始化项目 Issue；
- [ ] 初始化项目 Issue 不包含业务首页需求；
- [ ] 项目已经使用默认技术方案初始化；
- [ ] 项目可以启动；
- [ ] 我创建或准备创建了首页需求讨论 Issue；
- [ ] 首页需求 Issue 产出了最终规格和验证检查点；
- [ ] 我创建或准备创建了首页开发 Issue；
- [ ] 首页开发 Issue 引用了首页需求 Issue；
- [ ] 页面能看到“个人库存订单管理工具”；
- [ ] 页面能看到商品、库存、订单三个入口占位；
- [ ] 页面提示当前还没有接入真实数据；
- [ ] 本章没有实现登录、权限、多租户；
- [ ] 本章没有实现商品、库存、订单真实功能；
- [ ] 本章没有引入 API、数据库、本地存储或复杂路由；
- [ ] 我看过本章 diff，并大致知道哪些文件和初始化有关，哪些文件和首页有关。

## 自检问题

如果你能回答下面的问题，就可以进入下一章：

1. 初始化项目 Issue 和首页开发 Issue 有什么区别？
2. 为什么需求讨论可以单独成为一个 Issue？
3. 为什么开发 Issue 要引用需求 Issue？
4. 为什么本章不让你自己选择技术方案？
5. 页面能看到，是否等于功能已经完成？
6. 如果页面打不开，你应该把什么反馈给 AI？
7. 本章哪些检查点可以由人看页面完成？
8. 如果 AI 主动加入登录、权限、数据库或真实业务逻辑，为什么应该要求它回退？

```text
AI coding 不是一个大 prompt 做完所有事情，而是把初始化、需求讨论、开发实现和验证拆成可追踪的小任务。
```
