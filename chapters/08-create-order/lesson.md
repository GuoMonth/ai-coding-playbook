# 学习

本章训练从“单个对象”进入“对象关联”：订单不是孤立字段，它由多个商品明细组成。

## 1. 创建需求 Issue

复制下面的提示词给 AI：

```text
请通过 gh CLI 创建一个 GitHub Issue，用来讨论“创建订单”需求。

这个 Issue 只做需求讨论，不直接开发。

请在 Issue 中记录：

- 创建订单要解决什么问题；
- 一个订单如何选择多个商品；
- 订单明细最少包含哪些字段；
- 数量字段有什么校验；
- 创建成功后订单如何进入订单列表；
- 订单初始状态是什么；
- 不做提交订单、库存校验、库存扣减、退单、客户、支付、物流、API、后端数据库、登录、权限或多租户；
- 需要形成最终规格和验证检查点。

创建前请先给我看标题和正文，等我确认后再运行 gh 命令。
```

## 2. 收敛订单结构

让 AI 帮你理解订单和明细：

```text
请基于创建订单需求 Issue，先不要写代码。

请用非开发人员能理解的语言解释：

1. 订单和订单明细的区别；
2. 为什么一个订单可以包含多个商品；
3. 每一行明细最少需要哪些字段；
4. 数量校验如何设计；
5. 为什么本章只创建订单，不提交订单；
6. 本章明确不做的内容；
7. 验证检查点。
```

这里要特别看 AI 有没有把“创建订单”误解成“完成交易”。

## 3. 创建开发 Issue 并实现

需求确认后，再创建开发 Issue：

```text
请通过 gh CLI 创建一个 GitHub Issue，用来实现“创建订单”。

要求：

- 引用创建订单需求 Issue；
- 按已确认规格实现；
- 可以从已有商品中选择商品；
- 一个订单可以添加多个商品明细；
- 每行明细有商品和数量；
- 数量必须大于 0；
- 创建成功后订单出现在订单列表；
- 订单初始状态为草稿或待提交；
- 不做提交订单、库存校验、库存扣减、退单、客户、支付、物流、API、后端数据库、登录、权限或多租户。

创建前请先给我看标题和正文，等我确认后再运行 gh 命令。
```

确认后让 AI 执行：

```text
请根据创建订单开发 Issue 执行实现。

要求：

- 开发前先复述最终规格和验证检查点；
- 只改创建订单所需的最小文件；
- 不加入库存校验或库存扣减；
- 完成后运行项目或必要检查；
- 如果失败，先说明错误类别，再修复；
- 最后说明创建或修改了哪些文件。
```

## 4. 验证订单创建

重点验证多商品明细和不扣库存：

```text
请按创建订单 Issue 的验证检查点，带我验证：

- 是否可以选择已有商品；
- 是否可以添加多个商品明细；
- 数量为空、0 或负数时是否被阻止；
- 创建成功后订单是否出现在订单列表；
- 创建订单后库存是否保持不变；
- 是否没有出现提交、退单、客户、支付或物流能力。
```

## 5. 看 diff

让 AI 解释订单结构变化：

```text
请查看本章相关 diff，用非开发人员能理解的方式说明：

- 哪些改动和订单表单有关；
- 哪些改动和订单明细有关；
- 订单如何关联商品；
- 有没有引入库存校验或扣减；
- 为什么创建订单后库存不应该变化。
```

本章要建立的习惯是：一个功能名背后可能有多个动作，要先拆清楚动作边界。
