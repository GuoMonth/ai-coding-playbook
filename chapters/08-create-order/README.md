# 08. Create Order

本章目标是创建一个包含多个商品的订单。

完成本章后，你应该知道：

- 一个订单可以关联多个商品；
- 订单明细需要商品、规格和数量；
- 创建订单和提交订单是两个不同动作；
- 多行明细比单个表单更容易出错；
- 库存校验应该留到提交环节。

## 背景

上一章只展示订单列表。本章开始创建订单，但只创建一条订单记录，不扣库存。

这个阶段的重点是理解订单和商品之间的关联关系。

## 实现边界

本章只做创建订单的最小流程。

本章明确不做：

- 提交订单；
- 库存校验和库存扣减；
- 退单和库存恢复；
- 客户、支付、物流、发票；
- API、后端数据库、登录、权限、多租户；
- 复杂优惠、价格和税费。

## 本章最终结果

完成本章后，页面应该支持创建一条订单。最小流程包括：

- 从已有商品中选择商品；
- 为每个商品填写数量；
- 一个订单可以有多个商品明细；
- 数量必须大于 0；
- 创建成功后订单出现在订单列表；
- 订单状态可以是草稿或待提交。

本章不检查库存是否足够，也不扣减库存。
