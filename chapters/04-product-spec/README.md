# 04. Product Spec

本章目标是为商品增加最小规格信息。

完成本章后，你应该知道：

- “商品”可以继续拆出更具体的规格；
- 字段设计会影响后续功能；
- 规格需要名称、单位和约束；
- 早期可以用简化模型，但要知道它的限制；
- AI coding 前要先说清楚数据结构。

## 背景

前面两章只有商品本身。本章开始让商品有规格，例如同一个商品可以有不同规格或单位。

这里不追求完整 SKU 体系，只让学习者第一次接触“一个业务对象内部还有结构”的概念。

## 实现边界

本章只在商品上增加最小规格信息。

本章明确不做：

- 完整 SKU、条码、价格体系；
- 多级分类、供应商、图片；
- 入库、库存流水、订单；
- 数据持久化；
- API、数据库、登录、权限、多租户；
- 复杂规格组合和批量编辑。

## 本章最终结果

完成本章后，创建或展示商品时应该能看到最小规格信息。规格至少包含：

- 规格名称；
- 单位；
- 可选的规格编码或备注；
- 基本必填校验；
- 在商品列表或详情区域中展示。

规格数据可以继续保存在页面内临时状态中。
