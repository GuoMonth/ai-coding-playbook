# 18. Review And Knowledge Base

本章目标是复盘整个学习过程，把 AI coding 反馈沉淀成可复用知识。

完成本章后，你应该知道：

- 经验需要被整理，才会变成能力；
- Issue、规格、检查点、diff 和测试都可以进入复盘；
- 检查清单能降低重复犯错；
- 提示词应该从真实问题中沉淀；
- 项目规则能帮助后续 AI coding 保持边界。

## 背景

前面章节已经从首页做到商品、库存、订单、接口、调试、测试、重构、部署和进阶边界。

如果不复盘，这些经验会散落在对话、报错和临时修复里。本章让学习者把它们整理出来。

## 实现边界

本章聚焦课程收束和经验沉淀。

本章明确不做：

- 新增业务功能；
- 重写前面章节；
- 建设复杂文档系统；
- 替换现有工具；
- 做完整团队知识库治理；
- 深入 UI/UX、产品方法论或组织流程。

## 本章最终结果

完成本章后，学习者应该整理出几类材料：

- 常见 Issue 写法检查清单；
- 需求规格检查清单；
- 验证检查点模板；
- diff review 提示词；
- 测试失败反馈提示词；
- 范围扩张判断规则；
- 后续学习清单。

这些材料可以很短，但要来自课程中真实经历过的任务。
