# 09. Stock Check And Submit

本章目标是在提交订单时检查库存，并在库存足够时扣减库存。

完成本章后，你应该知道：

- 提交订单是会改变状态的业务动作；
- 库存不足时应该阻止提交；
- 库存足够时才可以扣减；
- 一次提交会同时影响订单状态和商品库存；
- 验证检查点必须覆盖成功和失败路径。

## 背景

上一章只创建订单，不改变库存。本章开始处理提交订单。

提交订单是第一个同时影响多个数据的动作：订单状态会变化，商品库存也会变化。

## 实现边界

本章只做库存校验和订单提交。

本章明确不做：

- 退单和库存恢复；
- 重复提交的完整幂等处理；
- 并发库存控制；
- 支付、客户、物流、发票；
- API、后端数据库、登录、权限、多租户；
- 复杂事务和分布式一致性。

## 本章最终结果

完成本章后，页面应该支持提交订单：

- 订单提交前检查每个商品库存；
- 库存不足时阻止提交并提示原因；
- 库存足够时订单状态变为已提交；
- 库存按订单明细数量扣减；
- 已提交订单不应再次作为普通草稿提交；
- 页面能看到订单状态和库存变化。

本章只建立库存校验和扣减直觉，复杂幂等和一致性留到后面。
