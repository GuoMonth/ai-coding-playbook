# 17. Advanced Auth And Tenant

本章目标是理解个人工具变成团队 SaaS 后，身份、权限和租户会带来什么复杂度。

完成本章后，你应该知道：

- 个人工具和团队 SaaS 的边界完全不同；
- 用户身份会影响谁能看见什么；
- 权限会影响谁能做什么；
- 多租户会影响数据属于谁；
- 这些能力重要，但不应该在入门主线过早实现。

## 背景

主线案例一直是个人库存订单管理工具。它默认只有一个人使用，所以不需要登录、权限和租户。

如果未来变成团队 SaaS，就必须重新思考身份、权限、组织和数据隔离。本章只建立边界意识。

## 实现边界

本章是进阶概念和评估，不要求改造主线系统。

本章明确不做：

- 生产级登录注册；
- RBAC 权限系统；
- 组织管理；
- 真实多租户隔离；
- 数据迁移；
- 安全合规方案；
- 重写前面所有章节。

## 本章最终结果

完成本章后，学习者应该能输出一份简短评估：

- 当前个人工具默认有哪些简化前提；
- 如果变成团队 SaaS，需要新增哪些概念；
- 身份、权限、多租户分别解决什么问题；
- 哪些改动会影响数据模型；
- 哪些能力应放到后续进阶，不进入当前主线；
- 为什么早期不应该让 AI 随手加入登录和权限。

本章重点是知道复杂度在哪里，而不是实现复杂系统。
