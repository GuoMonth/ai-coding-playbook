# 07. Order List

本章目标是做出订单列表页面。

完成本章后，你应该知道：

- 订单是一种可追踪记录；
- 订单状态会影响后续操作；
- 列表筛选是一种常见的查看方式；
- 展示订单和创建订单不是一回事；
- 订单页面要先有清晰边界，避免提前进入库存扣减。

## 背景

前面章节完成了商品、规格、入库和本地保存。本章开始引入订单模块。

第一步只做订单列表，用来理解订单编号、状态、明细摘要和可追踪性。

## 实现边界

本章只做订单列表展示和状态概念。

本章明确不做：

- 创建订单；
- 编辑订单明细；
- 提交订单和库存扣减；
- 退单和库存恢复；
- 支付、客户、物流；
- API、后端数据库、登录、权限、多租户；
- 复杂统计和报表。

## 本章最终结果

完成本章后，页面应该能看到订单列表入口和订单列表页面。订单列表至少包含：

- 订单编号；
- 订单状态；
- 商品数量或明细摘要；
- 创建时间；
- 空状态；
- 简单状态筛选。

订单数据可以先使用示例数据，或者展示空状态。
