# 学习

本章只训练一个目标：让 AI 按规格做出商品列表页面。

## 1. 创建需求 Issue

复制下面的提示词给 AI：

```text
请通过 gh CLI 创建一个 GitHub Issue，用来讨论“商品列表页面”需求。

这个 Issue 只做需求讨论，不直接开发。

请在 Issue 中记录：

- 商品列表页面要解决什么问题；
- 列表最少需要哪些字段；
- 空状态应该提示什么；
- 加载状态应该如何表现；
- 本章数据可以使用示例数据或页面内临时状态；
- 不做创建商品、商品规格、入库、订单、数据持久化、API、数据库、登录、权限或多租户；
- 需要形成最终规格和验证检查点。

创建前请先给我看标题和正文，等我确认后再运行 gh 命令。
```

这个 Issue 的重点不是写得很专业，而是让你意识到：列表也需要规格。

## 2. 收敛规格和检查点

让 AI 帮你把需求收敛到可验证状态：

```text
请基于商品列表需求 Issue，先不要写代码。

请帮我收敛：

1. 商品列表最小字段；
2. 空状态文案；
3. 加载状态或加载占位；
4. 从首页进入商品列表的最小方式；
5. 本章明确不做的内容；
6. 验证检查点。

请用非开发人员能理解的语言输出。
```

这里要特别检查 AI 有没有主动扩大范围。如果它建议“顺便做新增商品”“顺便接数据库”，要求它收回。

## 3. 创建开发 Issue 并实现

需求确认后，再让 AI 创建开发 Issue：

```text
请通过 gh CLI 创建一个 GitHub Issue，用来实现“商品列表页面”。

要求：

- 引用商品列表需求 Issue；
- 按已确认规格实现；
- 从首页能进入商品列表；
- 商品列表显示最小字段；
- 有空状态；
- 有简单加载状态或加载占位；
- 可以使用示例数据或页面内临时状态；
- 不做创建商品、商品规格、入库、订单、数据持久化、API、数据库、登录、权限或多租户。

创建前请先给我看标题和正文，等我确认后再运行 gh 命令。
```

确认后让 AI 执行：

```text
请根据商品列表开发 Issue 执行实现。

要求：

- 开发前先复述最终规格和验证检查点；
- 只改实现商品列表所需的最小文件；
- 不增加本章边界之外的功能；
- 完成后运行项目或必要检查；
- 如果失败，先说明错误类别，再修复；
- 最后说明创建或修改了哪些文件。
```

## 4. 验证页面

打开页面后，按检查点逐项看：

```text
请根据商品列表 Issue 的验证检查点，带我逐项验证页面。

如果某个检查点无法验证，请说明原因，并判断是规格没写清楚，还是实现没完成。
```

## 5. 看 diff

页面通过后，让 AI 解释改动：

```text
请查看本章相关 diff，用非开发人员能理解的方式说明：

- 哪些文件和商品列表页面有关；
- 哪些改动只是页面结构；
- 哪些改动是示例数据；
- 有没有超出本章边界的内容；
- 我现在不需要理解哪些代码细节。
```

本章要建立的习惯是：业务页面做出来后，也要检查它有没有偷偷变成更大的任务。
