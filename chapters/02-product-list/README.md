# 02. Product List

本章目标是做出第一个业务列表页面：商品列表。

完成本章后，你应该知道：

- 首页入口和业务页面可以分开实现；
- 列表页面需要明确字段、空状态和加载状态；
- 示例数据和真实数据不是一回事；
- Issue 需要写清楚“只展示，不新增、不保存、不入库”；
- 页面看起来像业务功能，不代表业务流程已经完整。

## 背景

上一章只做了首页占位。本章开始进入商品模块，但只做“看见商品列表”这一件事。

商品列表是后续创建商品、商品规格、入库和订单选择商品的基础。现在先用最小方式把列表结构跑通。

## 实现边界

本章只做商品列表展示。

本章明确不做：

- 创建商品表单；
- 商品规格编辑；
- 入库、扣库存、订单；
- 数据持久化；
- API、数据库、登录、权限、多租户；
- 复杂搜索、批量操作、导入导出。

## 本章最终结果

完成本章后，页面应该能看到商品列表入口和商品列表页面。商品列表至少包含：

- 商品名称；
- 商品编号或简称；
- 当前库存数量；
- 商品状态；
- 空状态提示；
- 简单加载状态或加载占位。

数据可以先用示例数据或页面内临时状态。刷新后丢失数据是允许的。
