# 验证

本章不检查代码功能，只检查你是否具备开始后续章节的最小意识。

## 完成清单

- [ ] 我知道本教程固定做“个人库存订单管理工具”；
- [ ] 我知道代码会放在 GitHub 仓库里；
- [ ] 我知道 Git 大概用来记录代码变化；
- [ ] 我知道 Issue 大概用来记录任务；
- [ ] 我知道可以让 AI 通过 `gh` CLI 创建 Issue；
- [ ] 我知道 Issue 写得清楚或模糊，会影响 AI coding 效果；
- [ ] 我知道规格和验证检查点能帮助 AI 判断自己是否做对；
- [ ] 我已经让 AI 尝试检查 `gh` CLI 是否可用；
- [ ] 我已经让 AI 创建或准备创建第一个 GitHub Issue。

## 自检问题

如果你能回答下面的问题，就可以进入下一章：

1. 本教程固定练习项目是什么？
2. 为什么本章不让你自己选择业务领域？
3. GitHub Issue 在 AI coding 里大概解决什么问题？
4. `gh` CLI 为什么适合交给 AI 使用？
5. commit、push、pull、branch 分别大概出现在什么环节？
6. 如果一个任务没有验证检查点，AI coding 会有什么风险？
7. 你是否知道自己还不需要完全掌握这些概念？

## 常见未完成状态

下面这些情况说明还不适合进入下一章：

- 你还不知道本教程接下来固定做什么项目；
- 你以为本章要开始写完整业务功能；
- 你完全不知道 Issue 和代码修改有什么关系；
- 你完全不知道 Git 会记录代码变化；
- 你认为 AI 写完代码后不需要检查；
- 你开始设计复杂权限、多租户或团队协作规则。

## 进入下一章前

进入下一章前，你只需要保留一个意识：

```text
AI coding 的任务最好被记录、被约束、可验证，并且能通过反馈不断修正。
```
