# 00. Getting Started

本章的目标不是开始写代码，而是建立后续所有章节都会重复使用的学习方式。

完成本章后，你应该拥有：

- 一个自己熟悉的业务场景；
- 一组可以替换默认示例的业务对象；
- 一个用于记录任务的 GitHub Issue 模板；
- 一份最小需求规格和验证检查点；
- 一套与 AI 协作的基本流程；
- 一个判断任务是否完成的最小方法。

## 本章为什么重要

AI coding 很容易让人直接进入“让 AI 写代码”的状态。但如果需求没有边界、任务没有验收标准、上下文没有记录，AI 生成的结果就很难验证，也很难沉淀经验。

本教程会反复训练同一个闭环：

```text
需求 -> 澄清 -> 任务 -> Issue -> 规格 -> 验证检查点 -> AI coding -> 反馈修正 -> 复盘
```

后续章节的业务任务会变化，但这个闭环会一直重复。重复本身就是训练的一部分。

AI coding 和传统开发的一个重要差别是：AI 可以在每一步实现后立刻运行测试、检查页面、读取错误信息，并根据反馈继续修正。为了让 AI 可靠工作，你不能只告诉它“做一个功能”，还要尽量告诉它“什么结果算对、用什么方式验证”。

本章只让你先感受这个思想：先写清楚最小规格和验证检查点，再让 AI coding 开始实现。后续章节会逐步把这种方式变成默认实践。

## 你需要准备什么

- 一个 GitHub 账号；
- 一个你可以操作的代码仓库；
- 一个 AI coding 工具；
- 一个你熟悉的业务领域。

你不需要先学会编程，也不需要理解所有软件工程概念。本章只要求你能描述自己的业务，并愿意用 GitHub Issue 记录任务。

## 默认示例和你的业务

本教程会使用“库存、订单、客户管理”作为默认示例。但这不是固定答案，只是为了让概念有一个共同参照。

你应该把它替换成自己的业务：

- 客户：你的业务中被服务、被管理或被交易的对象；
- 商品：你的业务中被展示、选择、购买、预约或使用的对象；
- 订单：你的业务中一次完整的申请、预约、交易、任务或流程；
- 库存：你的业务中会被占用、消耗、限制或统计的资源。

如果你暂时没有自己的业务，就先使用默认示例完成练习。
