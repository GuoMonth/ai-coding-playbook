# 复盘

完成本章后，请不要急着进入下一章。先用下面的问题整理你的理解。

## 业务场景复盘

1. 我选择的业务场景是什么？
2. 这个业务场景是不是我熟悉的？
3. 它是否足够小，可以从一个页面开始？
4. 它是否能逐步引出列表、表单、数据、流程、权限、错误和重构？

## GitHub Issue 复盘

回答下面的问题：

- 写 Issue 是否帮助我把想法变清楚了？
- 哪个部分最难写：背景、目标、范围还是验收标准？
- 我有没有写出“本次不做什么”？
- 我有没有写出最小需求规格？
- 我有没有写出能让 AI 自我检查的验证点？
- AI 对我的 Issue 提出了哪些修改建议？
- 下次创建 Issue 时，我会复用哪一段模板？

## 规格和测试反馈复盘

回答下面的问题：

- 如果 AI 完成了一个任务，我准备让它如何证明任务完成？
- 哪些结果可以通过页面截图或手动操作验证？
- 哪些结果未来适合变成自动化测试？
- 如果 AI 的实现失败，我会把什么反馈交还给 AI？
- 我是否理解“测试不是为了为难 AI，而是为了让 AI 更快修正”？
