# 自我验证

## 完成清单

- [ ] 我创建或准备创建了调试 Issue；
- [ ] 调试 Issue 写清楚了错误现象；
- [ ] 调试 Issue 写清楚了复现步骤；
- [ ] 调试 Issue 写清楚了期望结果和实际结果；
- [ ] 我让 AI 先诊断错误类型，而不是直接大改；
- [ ] 修复保持在最小范围；
- [ ] 修复后重新执行了原来的复现步骤；
- [ ] 修复后相关页面或流程仍然正常；
- [ ] 本章没有新增业务功能；
- [ ] 本章没有做大范围重构、性能优化或监控系统；
- [ ] 本章没有引入后端服务、真实数据库、登录、权限或多租户；
- [ ] 我把这次错误沉淀成了复盘或检查清单。

## 自检问题

如果你能回答下面的问题，就可以进入下一章：

1. 为什么复现步骤比一句“坏了”更重要？
2. 报错信息应该如何提供给 AI？
3. 什么叫最小修复？
4. 为什么修复后还要按原步骤重新验证？
5. 如果 AI 借机重构大量代码，应该怎么处理？
6. 这次错误属于哪一类问题？
7. 这次经验是否值得加入检查清单？
