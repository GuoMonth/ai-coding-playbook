# 12. Errors And Debugging

本章目标是练习面对错误：复现、定位、最小修复和验证。

完成本章后，你应该知道：

- 报错信息是给 AI 的重要反馈；
- 复现步骤比“它坏了”更有价值；
- 日志、页面现象和操作路径要一起描述；
- 修复应该尽量小，不要顺手重构；
- 修完后必须重新验证原来的场景。

## 背景

前面章节已经有多个页面和业务动作。功能变多后，错误会自然出现。

本章可以故意制造或选择一个小错误，用它训练调试过程，而不是追求复杂修复。

## 实现边界

本章只处理一个小而明确的错误。

本章明确不做：

- 新增业务功能；
- 大范围重构；
- 性能优化；
- 完整监控系统；
- 自动化测试体系；
- 后端服务、真实数据库、登录、权限、多租户；
- 一次性修复所有潜在问题。

## 本章最终结果

完成本章后，你应该得到一份清晰的调试记录：

- 错误现象；
- 复现步骤；
- 关键报错或日志；
- AI 对错误类型的判断；
- 最小修复方案；
- 修复后的验证结果；
- 本次经验沉淀。

本章重点是调试方法，不是错误本身。
