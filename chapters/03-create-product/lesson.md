# 学习

本章从商品列表推进到创建商品。重点不是表单多完整，而是让你看见“输入需要规则”。

## 1. 创建需求 Issue

复制下面的提示词给 AI：

```text
请通过 gh CLI 创建一个 GitHub Issue，用来讨论“创建商品表单”需求。

这个 Issue 只做需求讨论，不直接开发。

请在 Issue 中记录：

- 创建商品最少需要哪些字段；
- 哪些字段必填；
- 输入为空或不合法时如何提示；
- 点击保存时页面应该有什么状态；
- 保存成功后如何回到商品列表或更新列表；
- 数据可以先保存在页面内临时状态；
- 不做商品规格、图片、价格、入库、订单、数据持久化、API、数据库、登录、权限或多租户；
- 需要形成最终规格和验证检查点。

创建前请先给我看标题和正文，等我确认后再运行 gh 命令。
```

## 2. 收敛表单规格

让 AI 把表单变成可执行规格：

```text
请基于创建商品表单需求 Issue，先不要写代码。

请帮我收敛：

1. 最小字段；
2. 必填规则；
3. 错误提示文案；
4. 保存成功后的页面反馈；
5. 保存中状态；
6. 本章明确不做的内容；
7. 验证检查点。

请用非开发人员能理解的语言输出，并提醒我哪些地方容易让 AI 扩大范围。
```

如果 AI 直接建议接数据库或做完整商品资料，说明它把任务放大了。

## 3. 创建开发 Issue 并实现

需求确认后，再创建开发 Issue：

```text
请通过 gh CLI 创建一个 GitHub Issue，用来实现“创建商品表单”。

要求：

- 引用创建商品需求 Issue；
- 按已确认规格实现；
- 表单至少包含商品名称、商品编号或简称、初始状态；
- 必填项为空时要有错误提示；
- 保存时要有基本交互状态；
- 保存成功后能在商品列表看到新商品；
- 数据可以只保存在页面内临时状态；
- 不做商品规格、图片、价格、入库、订单、数据持久化、API、数据库、登录、权限或多租户。

创建前请先给我看标题和正文，等我确认后再运行 gh 命令。
```

确认后让 AI 执行：

```text
请根据创建商品开发 Issue 执行实现。

要求：

- 开发前先复述最终规格和验证检查点；
- 只改创建商品所需的最小文件；
- 不增加本章边界之外的字段和功能；
- 完成后运行项目或必要检查；
- 如果失败，先说明错误类别，再修复；
- 最后说明创建或修改了哪些文件。
```

## 4. 验证表单

重点验证正常路径和错误路径：

```text
请按创建商品 Issue 的验证检查点，带我验证：

- 必填为空时是否阻止保存；
- 错误提示是否可见；
- 输入合法时是否能保存；
- 保存后列表是否出现新商品；
- 刷新后数据丢失是否符合本章边界；
- 是否出现了本章不该有的功能。
```

## 5. 看 diff

最后让 AI 解释改动：

```text
请查看本章相关 diff，用非开发人员能理解的方式说明：

- 哪些改动和表单输入有关；
- 哪些改动和校验有关；
- 哪些改动和列表更新有关；
- 有没有引入数据持久化、API 或库存订单逻辑；
- 我现在不需要理解哪些代码细节。
```

本章要建立的习惯是：每个输入框背后都应该有规则、状态和验证方式。
