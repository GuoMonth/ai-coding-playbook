# 03. Create Product

本章目标是增加创建商品的最小表单。

完成本章后，你应该知道：

- 表单需要输入项、校验规则和错误提示；
- “能点击保存”不等于“保存规则正确”；
- 用户输入是软件最容易出错的地方之一；
- AI coding 需要先确认字段和校验，再实现页面；
- 页面内临时新增和真实数据保存仍然不是一回事。

## 背景

上一章只展示商品列表。本章让学习者创建商品，并把新商品显示到列表中。

这里的目标不是做完整商品管理，而是用一个很小的表单理解输入、校验和交互状态。

## 实现边界

本章只做新增商品的最小表单体验。

本章明确不做：

- 商品规格；
- 商品图片、价格、分类、供应商；
- 入库、库存流水、订单；
- 数据持久化；
- API、数据库、登录、权限、多租户；
- 复杂表单设计和批量导入。

## 本章最终结果

完成本章后，页面应该支持创建一个最小商品。表单至少包含：

- 商品名称；
- 商品编号或简称；
- 初始状态；
- 保存按钮；
- 必填校验；
- 保存中或提交中的交互状态；
- 错误提示。

新增商品可以只保存在页面内临时状态中，刷新后丢失是允许的。
