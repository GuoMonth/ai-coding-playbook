# 06. Data Persistence

本章目标是让商品、规格和库存数据在刷新后仍然存在。

完成本章后，你应该知道：

- 页面状态和保存下来的数据不是一回事；
- 持久化需要考虑记录、字段和恢复；
- 最小本地存储可以帮助 AI 获得快速反馈；
- 刷新页面是验证保存是否成功的直接方式；
- 本地保存不等于真正的后端数据库。

## 背景

前面章节的数据都可以只存在页面内。这样适合快速学习，但刷新后数据会丢失。

本章开始加入最小数据保存能力，让商品、规格和库存能在本地恢复。

## 实现边界

本章优先使用最简单的本地保存方式，例如浏览器本地存储。

本章明确不做：

- 后端服务和真实数据库；
- 用户账号、权限、多租户；
- 多设备同步；
- 数据导入导出；
- 数据迁移系统；
- 订单提交、退单、库存扣减；
- 复杂缓存和离线同步。

## 本章最终结果

完成本章后，页面应该支持：

- 创建商品后刷新页面仍然存在；
- 商品规格刷新后仍然存在；
- 入库后的库存数量刷新后仍然存在；
- 有一个清空本地数据或重置示例数据的入口；
- 保存失败时至少能给出可理解提示。

本章只要求理解最小持久化，不要求掌握数据库原理。
