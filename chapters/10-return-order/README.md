# 10. Return Order

本章目标是实现整单退回，并在退回后恢复库存。

完成本章后，你应该知道：

- 业务流程常常有正向和逆向；
- 退单不是删除订单；
- 退回会改变订单状态并恢复库存；
- 补偿动作也需要前置条件；
- 逆向流程要防止重复执行。

## 背景

上一章提交订单时扣减了库存。本章处理一个自然问题：订单退回后，库存应该恢复。

这里先做最简单的整单退回，不做复杂售后。

## 实现边界

本章只做整单退回和库存恢复。

本章明确不做：

- 部分退货；
- 退款、支付、售后审批；
- 换货、补发、物流；
- 重复退回的完整幂等处理；
- 并发库存控制；
- API、后端数据库、登录、权限、多租户；
- 复杂财务和审计系统。

## 本章最终结果

完成本章后，页面应该支持退回已提交订单：

- 只有已提交订单可以退回；
- 退回后订单状态变为已退回；
- 库存按订单明细数量恢复；
- 已退回订单不能再次普通退回；
- 页面能看到订单状态和库存变化；
- 不能通过退回来删除订单记录。

本章只建立补偿流程直觉，复杂售后留到更后面。
