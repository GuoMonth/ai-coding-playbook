# Day 2：数据、协作、前端与部署概念

Day 2 的目标是知道数据如何保存、代码如何协作、页面如何和后端连接，以及软件如何从本地走向可访问。

## 1. 数据库和 SQL

需要快速扫过这些概念：

- 数据库：专门保存和查询数据的系统；
- 表：同一类数据的集合；
- 行：一条记录；
- 列：一个字段；
- 主键：唯一标识一条记录；
- 外键：表达记录之间的关联；
- CRUD：新增、读取、更新、删除；
- SQL：操作关系型数据库的语言；
- Redis / MongoDB：常见非关系型数据系统。

提示词：

```text
请用“商品、库存、订单”这个例子解释数据库、表、行、列、主键、外键、CRUD 和 SQL。

请说明关系型数据库和 Redis、MongoDB 的区别，但不要展开底层原理。
```

你只需要记住：

```text
很多业务问题，最后都会落到记录、字段、关联和数据一致性上。
```

## 2. Git 和 GitHub 协作

需要快速扫过这些概念：

- 工作区：你正在修改的文件；
- 暂存区：准备提交的修改；
- commit：一次有说明的本地快照；
- branch：一条独立开发线；
- remote：远程仓库；
- push：把本地提交推到远程；
- pull：拉取远程更新；
- Pull Request：请求别人 review 并合并；
- conflict：两边修改冲突，需要人工判断。

提示词：

```text
请用“我让 AI 完成一个商品列表任务”的例子，解释工作区、暂存区、commit、branch、remote、push、pull、Pull Request 和 conflict。

请重点说明为什么 AI coding 后一定要看 diff。
```

你只需要记住：

```text
Git 不是开发者专属工具，它是你追踪 AI 改了什么、能否回退、能否 review 的基础。
```

## 3. 前端和 TypeScript

需要快速扫过这些概念：

- HTML：页面结构；
- CSS：页面样式；
- JavaScript：页面逻辑；
- TypeScript：带类型约束的 JavaScript；
- React：用组件组织页面；
- 组件：页面中可复用的一块；
- 状态：页面当前记住的数据；
- 事件：用户点击、输入等触发的动作。

提示词：

```text
请用“商品列表页面”和“创建商品表单”解释 HTML、CSS、JavaScript、TypeScript、React、组件、状态和事件。

请说明为什么类型、状态和事件会影响 AI coding 的质量。
```

你只需要记住：

```text
前端不只是好不好看，它还处理状态、输入、反馈和用户操作。
```

## 4. 全栈流转

用一条链路把前面概念串起来：

```text
用户点击按钮
-> 前端读取输入和状态
-> 发起 HTTP 请求
-> 后端处理业务规则
-> 数据库读取或保存记录
-> 后端返回 JSON
-> 前端更新页面
-> 用户看到结果或错误提示
```

提示词：

```text
请用“提交订单并扣减库存”的例子，解释完整全栈流转：

用户点击、前端状态、HTTP 请求、后端业务规则、数据库记录、JSON 响应、页面反馈。

请指出每一步可能出现什么错误，以及我应该如何反馈给 AI。
```

你只需要记住：

```text
当一个功能出问题时，不要只说“页面坏了”，要尽量定位它发生在前端、请求、后端、数据还是反馈环节。
```

## 5. 部署和云原生概念

需要知道这些概念存在：

- build：把项目构建成可发布产物；
- deploy：把软件发布到用户能访问的环境；
- environment：运行环境；
- Docker：用容器封装运行环境；
- image：容器镜像；
- container：运行中的容器；
- Docker Compose：在本地编排多个服务；
- CI/CD：自动检查、构建和发布；
- Kubernetes：大规模管理容器的系统。

提示词：

```text
请用“把一个库存订单工具发布给别人访问”的例子，解释 build、deploy、environment、Docker、image、container、Docker Compose、CI/CD 和 Kubernetes。

请特别说明哪些概念我现在只需要知道存在，不需要深入学习。
```

你只需要记住：

```text
部署问题经常不是代码逻辑错，而是环境、配置、构建或运行方式不一致。
```

如果你想在 2 天结束时做一个小成品，可以选择一个临时工具，不要求和主线课程一致。

```text
- 返回一段 JSON；

```

提示词：

```text

- 前端可以非常简单；
- 后端只保留一个接口；
```

这个成品不是主线项目，只是帮助你把概念串成一次体验。
