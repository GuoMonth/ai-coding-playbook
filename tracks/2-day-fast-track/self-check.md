# 自我验证

这份自检不考代码。它只检查你是否建立了软件开发概念地图。

## 完成清单

- [ ] 我能用一句话解释 CPU、内存、磁盘和操作系统的关系；
- [ ] 我知道 JSON 为什么常出现在 API 和 AI coding 中；
- [ ] 我知道浏览器打开网页时会经过域名、DNS、IP、端口和 HTTP；
- [ ] 我能区分 404 和 500 的大致含义；
- [ ] 我知道变量、类型、条件、循环、函数和异常分别大概做什么；
- [ ] 我知道数据库中的表、行、列、主键、外键是什么；
- [ ] 我知道 CRUD 表示新增、读取、更新和删除；
- [ ] 我知道 Git 的 commit、branch、push、pull、Pull Request 分别解决什么问题；
- [ ] 我知道前端不只是视觉，还包括状态、事件和反馈；
- [ ] 我能说出一次全栈请求从用户点击到页面反馈的大致链路；
- [ ] 我知道 build、deploy、Docker、CI/CD、Kubernetes 分别处于部署概念中的什么位置；
- [ ] 我知道哪些概念现在只需要知道存在，不需要深入细节。

## 自检问题

如果你能回答下面的问题，就说明这条速通路线达到了目标：

1. 为什么软件运行会涉及 CPU、内存、磁盘和操作系统？
2. 为什么 API 经常返回 JSON？
3. 一个网页打不开时，可能是网络、前端、后端还是数据问题？
4. 状态码 404 和 500 分别更可能说明什么？
5. 为什么数据库需要主键？
6. 商品和订单之间为什么可能需要外键或关联关系？
7. 为什么 AI coding 后要看 diff？
8. 前端状态和后端数据有什么区别？
9. “提交订单并扣减库存”这件事，在全栈链路中会经过哪些环节？
10. 为什么部署问题不一定是代码逻辑问题？
11. Docker 和 Kubernetes 为什么不是入门阶段必须深究的内容？
12. 你现在最容易混淆的 3 个概念是什么？

## 让 AI 严厉检查

完成自检后，把下面的提示词交给 AI：

```text
请根据我刚完成的“2 天 AI Coding 概念速通”，对我进行一次严格口试。

要求：

- 一次只问一个问题；
- 如果我回答太空泛，请继续追问；
- 不考代码语法；
- 重点考我是否能说清概念位置、解决的问题和边界；
- 最后给我一份薄弱概念清单，并建议我接下来进入主线课程时优先注意什么。
```

## 结束标准

这条路线结束时，你不需要会写软件。

你只需要能做到：

```text
听到一个软件开发名词时，知道它大概属于哪一层，解决什么问题，以及该如何继续向 AI 追问。
```
