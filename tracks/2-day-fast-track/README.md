# 2 天 AI Coding 概念速通

这是一条概念地图路线。它不替代主线课程，也不承诺 2 天掌握开发能力。

它的目标是：让非开发角色快速知道软件开发世界里有哪些关键概念、它们大致处在系统的什么位置，以及后续如何继续向 AI 追问。

## 适合谁

- 想先快速扫一遍软件开发全貌的人；
- 面对 CPU、HTTP、SQL、Git、React、Docker 等名词容易失去方向的人；
- 还没准备进入主线实战，但希望先建立系统观的人；
- 希望用 AI 做伴学、追问和自测的人。

## 速通方法论

这条路线采用四步学习法：

1. **先定位概念**：先问它是什么、解决什么问题、处在系统哪一层；
2. **用 AI 伴学**：让 AI 按你的职业背景解释、举例、对比，也可以组织正反观点辩论；
3. **主动放弃细节**：明确现在不深究什么，避免被底层细节和框架差异拖住；
4. **用场景验证**：用小场景、极简成品和自测题确认自己是否真的理解。

遇到概念时，不要直接查长文章。先把概念交给 AI，用下面的问题追问：

```text
我是非开发角色。请用我能理解的语言解释 [概念]：

1. 它是什么；
2. 它解决什么问题；
3. 它在一个软件系统中处于什么位置；
4. 它在 AI coding 中什么时候会影响我；
5. 请给一个极简例子；
6. 请告诉我现在不需要深究哪些细节。
```

每完成一个阶段，可以让 AI 做一次严厉自测：

```text
请扮演严厉的软件工程老师，根据我刚学的内容出 8 道题。

要求：

- 题目面向非开发角色；
- 不考代码细节；
- 重点考我是否理解概念位置、解决的问题和边界；
- 我回答后，请指出我哪里理解浅了。
```

如果你想加深理解，可以让 AI 做一次正反辩论：

```text
请围绕 [概念] 组织一次正反辩论。

背景：

- 我是非开发角色；
- 我只需要理解这个概念在软件系统中的位置和边界；
- 不讨论复杂实现细节。

要求：

- 正方说明它为什么重要；
- 反方说明入门阶段为什么不该过度学习它；
- 最后给我一个平衡结论：现在应该掌握什么，暂时可以放下什么。
```

本路线只要求建立概念位置感，不要求掌握编码能力、搭建生产系统或理解底层细节。

## 和主线课程的关系

主线课程训练的是 AI coding 中的工程判断和反馈闭环。

这条速通路线训练的是软件开发世界的概念地图。你可以先走速通路线，再进入主线课程；也可以在主线课程中遇到陌生概念时，回来查这条路线。
