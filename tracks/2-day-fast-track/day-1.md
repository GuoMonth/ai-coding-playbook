# Day 1：底层心智、网络与后端逻辑

Day 1 的目标不是学会编程，而是知道软件为什么能运行、数据如何表达、网络如何连接，以及后端逻辑大概是什么。

## 1. 计算机如何运行

先建立最小心智模型：

- CPU：负责计算；
- 内存：负责临时保存正在运行的数据；
- 磁盘：负责长期保存文件和数据；
- 操作系统：负责管理硬件、进程、文件和网络；
- 程序：运行在操作系统上的一组指令；
- 进程：正在运行的程序实例。

提示词：

```text
请用“一个软件从点击启动到运行起来”的过程，解释 CPU、内存、磁盘、操作系统、程序和进程之间的关系。

不要讲底层电路细节，只讲我在 AI coding 中需要理解的部分。
```

你只需要记住：

```text
软件运行时，会占用计算资源、内存资源、文件资源和网络资源。
```

## 2. 数据如何表达

需要快速扫过这些概念：

- bit / byte：计算机保存数据的最小单位；
- 字符编码：让文字能被计算机保存和传输；
- UTF-8：现代软件常见字符编码；
- JSON：前后端、API 和 AI 工具常见的数据交换格式；
- 文件：长期保存数据的一种形式。

提示词：

```text
请用“一个商品订单被保存和传输”的例子，解释 bit、byte、UTF-8、JSON 和文件。

重点说明 JSON 为什么在 API 和 AI coding 中经常出现。
```

你只需要记住：

```text
很多软件问题，本质上是数据格式、字段含义或编码方式没有说清楚。
```

## 3. 网络如何连接

需要快速扫过这些概念：

- IP：网络地址；
- 域名：更容易记住的地址名称；
- DNS：把域名解析成 IP；
- 端口：同一台机器上不同服务的入口；
- TCP/IP：保证数据可靠传输的一组协议；
- HTTP：浏览器、前端和后端常用的请求响应协议；
- GET / POST：常见请求方式；
- 状态码：请求结果的简短编号，例如 200、404、500；
- HTTPS：加密的 HTTP。

提示词：

```text
请用“浏览器打开一个网页”的过程，解释域名、DNS、IP、端口、TCP/IP、HTTP、GET、POST、状态码和 HTTPS。

请特别说明 404 和 500 分别意味着什么，以及我应该如何把这些信息反馈给 AI。
```

你只需要记住：

```text
页面打不开、接口失败、状态码异常，都是 AI coding 中非常重要的反馈。
```

## 4. 编程最小概念

不需要系统学习编程语言，但要知道这些词：

- 变量：给数据取名字；
- 类型：数据的类别；
- 列表：一组数据；
- 字典 / 对象：带字段名的数据；
- 条件判断：根据情况走不同路径；
- 循环：重复处理一组数据；
- 函数：把一段逻辑封装成可复用动作；
- 异常：程序运行中出现的错误。

提示词：

```text
请用“提交订单并扣减库存”的例子，解释变量、类型、列表、对象、条件判断、循环、函数和异常。

不要让我写复杂代码，只给极简伪代码和业务解释。
```

你只需要记住：

```text
代码不是神秘文字，它主要是在表达数据、条件、流程和异常处理。
```

## 5. 第一个后端逻辑体验

如果你有精力，可以让 AI 用 Python + FastAPI 做一个极简接口。

这个体验只用于理解后端逻辑，不进入主线课程，也不要求长期维护。

提示词：

```text
请带我在一个临时目录里创建一个极简 FastAPI 示例。

目标：

- 提供一个 GET 接口；
- 浏览器访问后返回一段 JSON；
- JSON 中包含 message 和 status 字段；
- 帮我解释每个文件的作用；
- 帮我运行并验证接口能访问；
- 不接数据库；
- 不做登录；
- 不做复杂项目结构。
```

完成后让 AI 解释：

```text
请用非开发人员能理解的语言解释：

- 浏览器访问接口时发生了什么；
- FastAPI 在这里扮演什么角色；
- JSON 响应是什么；
- 如果接口打不开，我应该收集哪些反馈给 AI。
```
