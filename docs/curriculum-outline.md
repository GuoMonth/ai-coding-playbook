# 课程大纲

本教程的目标不是教授完整的软件开发技能，而是用一个固定、极简的业务软件，训练产品经理、UI/UX 设计师等非开发角色在 AI coding 中需要的工程意识。

## 总体定位

教程固定围绕一个简单的“客户与商品管理工具”展开。

选择这个场景的原因很简单：

- 大多数人都能理解客户和商品；
- 可以很快做出可见页面；
- 可以自然引出列表、表单、数据、接口、错误、测试、重构等概念；
- 不需要学习者一开始就设计自己的业务。

本教程早期不会要求学习者把客户、商品、订单映射到自己的行业。先把固定场景走通，理解 AI coding 的工作方式，再在后续复盘中讨论如何迁移到自己的业务。

## 教学边界

本教程不做概念百科，也不把 Git、GitHub、Issue、测试、部署讲成独立工具课。

每个概念只做三件事：

1. 让学习者知道它存在；
2. 给出最小可用解释；
3. 提供提示词，让学习者和 AI 继续讨论、辩论和自检。

很多概念不需要在第一次出现时讲透。学习者会在后续任务中反复遇到它们，通过真实反馈逐步理解。

## 学习闭环

后续每个任务尽量走同一套闭环：

1. 用自然语言描述一个小需求；
2. 让 AI 帮忙澄清任务边界；
3. 让 AI 通过 `gh` CLI 创建或更新 GitHub Issue；
4. 在 Issue 中保留最小规格和验证检查点；
5. 让 AI coding 实现任务；
6. 运行页面、命令或测试，让 AI 获得反馈；
7. 根据反馈修正；
8. 查看 diff，理解这次改了什么；
9. commit、push，并在需要时发起 PR；
10. 复盘本次任务学到的工程意识。

这里的重点不是让学习者背命令，而是让学习者知道：AI coding 不是一次性生成代码，而是在仓库、Issue、规格、测试反馈和版本记录中持续迭代。

## 固定业务主线

教程默认从一个非常小的工具开始：

```text
客户与商品管理工具
```

前期只做：

- 查看客户；
- 创建客户；
- 查看商品；
- 创建商品。

中期再逐步加入：

- 数据保存；
- 简单订单；
- 库存数量；
- API；
- 错误处理；
- 测试。

后期再引入：

- 权限和数据范围；
- 幂等和一致性；
- 重构；
- 部署；
- 复盘和知识库。

## 章节结构

每章建议保持固定文件结构：

- `README.md`：本章目标和学习方式；
- `task.md`：本章要完成的固定任务；
- `prompts.md`：给 AI 的提示词；
- `concepts.md`：只做最小概念提醒；
- `verify.md`：完成检查和自检点；
- `reflection.md`：复盘问题和经验沉淀。

## 大纲草案

### 00. Getting Started

建立最小 AI coding 工作方式。知道本教程固定做客户与商品管理工具，知道 Git/GitHub/Issue/gh/规格/验证检查点的存在，并让 AI 通过 `gh` CLI 创建第一个 Issue。

重点意识：仓库、Issue、gh CLI、Git、commit、push、pull、分支、PR、规格、验证检查点、测试反馈。

### 01. First Visible Page

做出第一个可见页面，让学习者看到软件可以被运行和观察。

默认任务：创建客户与商品管理工具的首页或工作台。

重点意识：项目、页面、组件、运行、预览、可见反馈。

### 02. Customer List

做客户列表页面。

重点意识：列表、字段、展示数据、空状态、加载状态。

### 03. Create Customer

增加创建客户的表单。

重点意识：表单、输入、校验、错误提示、交互状态。

### 04. Product List

做商品列表页面。

重点意识：业务记录、字段差异、复用、重复代码。

### 05. Create Product

增加创建商品的表单。

重点意识：相似需求、复制粘贴风险、组件复用的第一层直觉。

### 06. Data Persistence

让客户和商品数据在刷新后仍然存在。

重点意识：本地存储、数据库、数据模型、记录、字段。

### 07. Simple Order Flow

引入一个最简单的订单流程，把客户和商品关联起来。

重点意识：业务流程、状态、前后步骤依赖、完成条件。

### 08. API And Contracts

把页面和数据服务分开，理解前后端协作。

重点意识：API、请求、响应、接口契约、字段匹配、错误返回。

### 09. Errors And Debugging

故意面对错误，而不是只让 AI 反复改。

重点意识：报错、日志、复现、定位、最小修改、验证修复。

### 10. Testing And Regression

开始让 AI 写或运行简单测试。

重点意识：测试、回归、自动化检查、验证反馈、改动影响范围。

### 11. Auth And Scope

引入登录、角色或数据范围的最小概念。

重点意识：身份、权限、数据范围、最小租户意识。

### 12. Consistency And Idempotency

引入重复提交、重复下单、库存扣减等问题。

重点意识：幂等、一致性、边界场景、并发的最小直觉。

### 13. Refactoring

通过新增需求暴露早期简单实现的局限，理解为什么需要重构。

重点意识：重构、技术债、职责边界、简单实现和长期维护的取舍。

### 14. Deploy And Operate

让软件从本地运行走向可访问。

重点意识：部署、环境变量、配置、发布、运行后反馈。

### 15. Review And Knowledge Base

复盘整个学习过程，把 AI coding 反馈沉淀成个人或团队知识库。

重点意识：代码审查、任务复盘、检查清单、提示词模板、工程判断规则。

## 组织原则

前期优先给视觉反馈，让学习者能看到页面和交互；中期引入数据、流程和接口；后期进入错误处理、测试、幂等、重构和部署等更抽象的工程判断。

本教程固定业务场景，是为了降低初学门槛。真正要训练的是 AI coding 时代的工程意识，而不是客户与商品管理这个业务本身。
